A Solução? 🤔

Precisamos de um lugar para "anotar" as pontuações... que não seja o nosso computador.

Precisamos de um...
CADERNO MÁGICO NA INTERNET! ☁️📓

O que é um "Banco de Dados"?

É um "guarda-volumes" digital na internet.

É um lugar seguro para guardar informações (como nomes, pontos, vidas, etc.) e que qualquer um pode aceder (se tiver a chave!).

Conheçam o... FIREBASE! 🔥

É uma SUPER caixa de ferramentas do Google para programadores.

E tem o "Banco de Dados" perfeito para o nosso jogo!

Como funciona? 🧐

Pense no Firebase como Armários Mágicos na escola:

🧑‍💻

Cada Jogador ("Ana", "Bruno") é um armário.

🔢

A Pontuação ("50", "42") é o que guardamos lá dentro.

O Banco de Dados "em Tempo Real"

A parte mais fixe? Quando um jogador salva uma pontuação nova...

...o ranking de TODOS os outros jogadores atualiza SOZINHO, na mesma hora! ⏱️✨

DESAFIO: Ligar o Jogo! 🚀

Temos 4 Passos para ligar o nosso jogo "Flappy Robô" 🤖 ao Firebase 🔥.

Passo 1: Criar o "Caderno" 📓

Primeiro, temos de ir ao site do Firebase e criar um "projeto" novo.

É como pegar num caderno novo e escrever "PONTUAÇÕES" na capa.

(Isto é feito no site do Google, não no código).

Passo 2: As Chaves Secretas 🔑

O Firebase dá-nos umas "chaves secretas" (um código de configuração).

Este código diz ao nosso jogo HTML exatamente ONDE, no mundo inteiro, está o NOSSO caderno.

const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  databaseURL: "..."
};

Passo 3: Chamar o "Carteiro" 💌

Temos de adicionar "Scripts" (os ajudantes) ao nosso HTML.

É como pôr um "Carteiro Mágico" 💌 no nosso jogo, que sabe como levar e trazer cartas (pontos) do Firebase.

<script src="...firebase-app.js"></script>
<script src="...firebase-database.js"></script>

Passo 4: Ligar o Jogo! 🔌

No nosso JavaScript, usamos as "Chaves Secretas" 🔑 para ligar o "Carteiro" 💌 ao nosso "Caderno" 📓.

A partir daqui, a magia acontece!

// Iniciar o Firebase
firebase.initializeApp(firebaseConfig);
// Pegar o nosso banco de dados
const db = firebase.database();

Como Salvar a Pontuação? 💾

Quando o jogo acaba (`gameOver`), temos de:

1. Perguntar o nome do jogador.
2. Mandar o "Carteiro" guardar!

let nome = prompt("Perdeu! Qual seu nome?");
if (nome) {
  // Comando para SALVAR no armário "nome"
  db.ref('scores/' + nome).set(pontuacao);
}

E o HANKING (Ranking)? 🏆

Salvar é bom, mas queremos VER quem são os melhores!

Precisamos de um lugar no HTML para mostrar a lista.

<div id="ranking">
  <h2>🏆 MELHORES 🏆</h2>
  <ul id="lista-ranking"></ul>
</div>

Como LER o Ranking? 📖

Dizemos ao Firebase: "Fica de olho em TODOS os armários 🗄️ e me avisa sempre que algo mudar!"

// Comando para LER todos os 'scores'
db.ref('scores').on('value', (snapshot) => {
  // "snapshot" tem todos os nomes e pontos!
  atualizarRankingNaTela(snapshot.val());
});

O Jogo Completo! 🤩

Agora o nosso jogo "Flappy Robô" tem:

  • O Jogo (HTML/CSS/JS)
  • Um Banco de Dados (Firebase)
  • Um Ranking em TEMPO REAL! 🏆

Mas... só funciona no NOSSO PC!

Como é que eu mando o jogo para o meu amigo jogar?

Não podemos mandar o ficheiro HTML... precisamos de um SITE!

O GRANDE DESAFIO FINAL! 🚀

Vamos colocar o nosso Jogo na INTERNET para todo o MUNDO jogar!

🌐

Ferramenta 1: GITHUB 🐙

O GitHub (o gatinho-polvo) é como uma Mochila Mágica 🎒 para guardar código.

É o lugar onde todos os programadores do mundo guardam seus projetos para não perderem.

Ferramenta 2: GITHUB PAGES 📜

O "GitHub Pages" é o Palco Mágico 🎭 do GitHub.

Você aperta um botão, e o GitHub transforma o seu ficheiro `index.html` num SITE de verdade, com um link público!

Os 3 Passos Finais!

1️⃣

Criar o Projeto no GitHub e enviar o(s) ficheiro(s) do jogo.

2️⃣

Ir a "Settings" (Definições) ⚙️ e encontrar a secção "Pages".

3️⃣

Ativar! O GitHub dá-lhe um link (ex: `seunome.github.io/meujogo`).

A MÁGICA COMPLETA! ✨

Agora, você pode mandar esse LINK para qualquer pessoa no mundo!

Todos que jogarem no seu link...
...vão ver o MESMO RANKING, ao MESMO TEMPO!
